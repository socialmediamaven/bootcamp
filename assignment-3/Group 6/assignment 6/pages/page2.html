Techincal Specification for Number Application

1. Write a function that prompts the user for a lower bound number.

a. If the number is correct then it should prompt again asking for the upper bound number.

b. If it is NAN then it should re-prompt until a number is entered.

2. Write a function that prompts a user for a higher bound number.

a.) If it is correct it should then print to the screen a message that includes the lower bound and higher bound number, as well as the number between it. Notifying the user that they got the correct number.

b.) If it is incorrect meaning the number is outside the bound of the lower number and higher number, then it should print to the screen that they got the incorrect. Stating that there solution was outside of the defined parameters.

3. Write a function that will test to see if the text entered is a string or number and if it is a string return the prompt box until a number is submitted.

a. The test case should test to make sure that the user inputs a low number and a high number and one in between. If this is correct the prompt box should then close and a message should be printed to the screen stating the two numbers they picked and the number in between.

b. it should also test for a number entered that is outside the defined parameter. (i.e. if the user enters five as there lower bound number and nine as there upper bound number, but picks 2 as their in between number. Then it should print out that they have entered an incorrect submission.)

4. This application should be written in javascript. Which should be used to update the view and write the programs logic.

5. HTML should be used to show the button and text.

6. CSS should be used for styling the page.
